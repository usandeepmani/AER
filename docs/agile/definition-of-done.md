# ✅ Definition of Done (DoD)

A task is considered **Done** only when it satisfies all applicable criteria.

---

# 📚 Learning Tasks

A Learning task is complete only if:

- [ ] Learning material completed
- [ ] Notes added to AER
- [ ] Able to explain the concept in your own words
- [ ] Relevant code written (if applicable)
- [ ] GitHub Issue moved to Done

Artifact(s):

- Notes
- Code example (optional)

---

# 💻 Coding Tasks

A Coding task is complete only if:

- [ ] Code implemented
- [ ] Code committed to GitHub
- [ ] Code follows project standards
- [ ] Documentation updated (if required)
- [ ] GitHub Issue moved to Done

Artifact(s):

- Source code
- Commit

---

# 🚀 Project Tasks

A Project task is complete only if:

- [ ] Project implementation completed
- [ ] README updated
- [ ] Code committed
- [ ] Project added to `/projects`
- [ ] Lessons learned documented
- [ ] GitHub Issue moved to Done

Artifact(s):

- Project
- README
- Documentation

---

# 🔍 Review Tasks

A Review task is complete only if:

- [ ] Sprint review completed
- [ ] Retrospective written
- [ ] Action items captured

Artifact(s):

- Retrospective
- Improvement actions
